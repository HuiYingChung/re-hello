這個就**很像真正的 app prototype 了，而且比前一份強很多**。
前一份比較像策略展示，這一份已經明顯進入 **產品互動原型** 的層級。

我的整體評價是：

**方向對、氣質對、互動也開始對了。**
如果這是你要拿來驗證概念、做 portfolio case study、或繼續往 MVP 實作推進，我會說這份比前一版成熟很多。

## 這份 prototype 最好的地方

### 1. 已經有真正的產品主線

這次不只是講概念，而是已經有幾條明確 flow：

* Home
* Add Person
* Person Detail
* Recall Card
* Event Prep

尤其 **Add → Detail → Recall** 這條線已經很清楚，使用者會怎麼用，現在看得出來了。

### 2. 語氣非常一致

這份最成功的地方之一，是文案真的有做到你想要的那種：

* 不像 CRM
* 不像商務工具
* 溫柔
* 對內向者友善

像：

* “Remember anyone new?”
* “Remember someone”
* “You don’t need to talk to everyone.”
* “One genuine conversation is worth more than ten business cards.”

這些都很有產品 personality，而且跟你原本的核心想法很一致。

### 3. Add Person 做得比一般表單好

我很喜歡它不是傳統硬表單，而是 **conversational step-by-step**。
像：

* What’s their name?
* What was your first impression?
* What did you two talk about?
* Next time you see them, what could you ask?

這比直接丟一堆欄位更符合你的產品定位，也更像在幫使用者「回想人」，不是在建資料庫。

### 4. Recall Card 很有特色

Recall Card 是這個 prototype 裡最有產品辨識度的部分。
它把資訊濃縮成：

* You met
* You talked about
* Remember this
* You could ask

這非常符合你之前一直講的核心價值：
**幫助內向的人在下次見面前更有準備。** 

### 5. 視覺風格是對的

整體視覺有這些優點：

* 溫暖色調
* 不商業化
* 有留白
* 卡片感舒服
* typography 有 personality
* 不像冷硬 dashboard

這很重要，因為如果 UI 長得像 CRM，再好的概念都會被拉歪。這版沒有這個問題。

## 我覺得還可以再改進的地方

### 1. Event Prep 有點搶戲

Event Prep 本身做得不差，而且很貼心。
但如果從產品聚焦來看，它現在有點把 app 拉向「社交教練 / 社交陪伴工具」，而不只是「social memory app」。

這不是錯，但你要決定第一版的主角到底是：

* **記錄與回顧**
  還是
* **社交前準備 + 記錄 + 回顧**

我還是傾向前者比較強。
所以如果是 MVP，我會建議：

**Event Prep 可以保留，但不要當主功能。**

### 2. Bottom nav 有幾個項目還沒真的成立

底部有：

* Home
* People
* Add
* Reminders
* Profile

但目前實際上主要完成的是 Home / Add，People、Reminders、Profile 還比較像 placeholder。

這對 prototype 沒關係，但如果你要拿去展示，我會建議：

* 要嘛把它們做成可進入的簡單頁
* 要嘛先縮成比較誠實的 nav

不然會讓人有一點「看起來完整，但其實還沒做」的感覺。

### 3. Add flow 的 skip 邏輯要再想一下

現在 step-based input 很好，但有些題目如果被 skip，之後 recall card 的內容可能會變薄。

所以你之後可能要想：

* 哪些欄位是必要的
* 哪些欄位可以空白
* 空白時 recall card 怎麼優雅顯示，不要破版或太空

### 4. Home 現在有點像「展示狀態」，還不夠像日常使用首頁

目前 Home 視覺很好，但它有點偏 demo 狀態：
Recent people、Review before you meet again、Upcoming 都合理，可是長期使用時還要再想資訊優先順序。

例如未來你可能要決定首頁最重要的是：

* 今天要回顧誰
* 最近新增誰
* 哪些待 follow-up
* 搜尋入口
* 快速新增

現在已經有雛形了，但還可以再 sharpen。

## 我最建議你下一步做什麼

現在最值得做的不是大改，而是這三件事：

1. **決定 MVP 主線是不是只鎖定**

   * Add Person
   * Person Detail
   * Recall Card
   * Reminder

2. **把 People / Reminders 頁補成最低限度可用**

3. **做一輪 usability thinking**
   想一想：

   * 第一次打開的人懂不懂這 app 在幹嘛
   * 新增一個人會不會太長
   * Recall Card 是不是真的 10 秒可看懂
   * 使用者會不會願意一直記錄
