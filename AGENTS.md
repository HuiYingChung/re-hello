# Repository Guidance

Before planning, editing, testing, publishing, or deploying this repository:

1. Read [`lessons.md`](lessons.md) completely.
2. Apply every relevant prevention rule from that file before repeating the associated operation.
3. Read the closest nested `AGENTS.md`; rules closer to a file add to or override this repository-level guidance.
4. Record every new failed attempt honestly in `lessons.md` with its evidence, root cause, recovery, and a concrete prevention rule before declaring the task complete.
5. Do not delete, soften, or rewrite an earlier failure to make a later outcome look cleaner. Add a dated correction when new evidence changes the diagnosis.

Engineering logs and decision records must remain in English and must distinguish local verification, exact-commit CI, deployment, and production evidence.
